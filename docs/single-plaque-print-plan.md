# 单张牌位套打模块重构与文档落地计划

## Summary

单张牌位套打模块第一阶段正式收敛为两类模板：

- `延生禄位模板`
- `往生牌位模板`

模板设计器从现有“混合模板结构”改为“模板类型 + 模板实例 + 版式”的结构。切换版式时，字段映射、画布布局、静态字段显示范围与保存上下文同步切换。

第一阶段同时完成以下配套改造：

- 新建牌位表单补齐 `属相 / 年龄 / 生日 / 忌日 / 寄语`
- 单张牌位导入后按记录自动判定 `__variant`
- 右侧数据表按当前导入数据类型切换字段列

后续实现、测试、回归均以本文件为执行基准。

## Phase 1 Scope

### 正式实施

- `延生禄位模板`
- `往生牌位模板`

### 仅预留扩展位，不进入第一阶段

- `延生牒壳模板`
- `延生牒芯模板`
- `往生牒壳模板`
- `往生牒芯模板`

## 单张牌位模板结构

### 1. 延生禄位模板

支持新增模板实例。

固定版式：

- `版式一`：`谢菩萨`
- `版式二`：`个人祈福禄位`

规则：

- 切换版式时，下方字段自动切换
- 切换版式时，画布字段和布局一起切换
- 每个版式独立保存

自动判定规则：

- `信人` 有值 -> `版式一`
- `信人` 空值 -> `版式二`

### 2. 往生牌位模板

支持新增模板实例。

固定版式：

- `版式一`：`无记名牌位`
- `版式二`：`单亡者`
- `版式三`：`二名亡者`

规则：

- 切换版式时，下方字段自动切换
- 切换版式时，画布字段和布局一起切换
- 每个版式独立保存

自动判定规则：

- `2 个亡者` -> `版式三`
- `1 个亡者` 且 `年龄 / 生日 / 忌日` 全空 -> `版式一`
- `1 个亡者` 且以上任一有值 -> `版式二`

说明：

- 第一阶段往生版式自动判定**不使用**`寄语`
- 但录入端和右侧数据表**保留**`寄语`

## 模板设计器改造目标

现有设计器从“混合模板结构”改为“模板类型 + 版式”的结构。

### 当前上下文

单张牌位设计器只看三个上下文：

- 当前模板类型
- 当前模板
- 当前版式

### 结构要求

- 去掉历史兼容式 `variant alias` 主流程依赖
- 每个模板自带自己的 `variants[]`
- 单张牌位保存粒度固定为：
  - `templateId`
  - `variantKey`
- 不再混用 `tabletType` 和模板 id 作为布局键

### 切换版式时必须同步切换

- 字段映射区
- 画布动态字段
- 静态字段显示范围
- 当前版式布局

## 右侧数据表切换规则

右侧数据表必须绑定当前导入数据类型。

### 导入延生数据时显示

- `牌位主体`
- `信人`
- `年龄`
- `属相`
- `生日`
- `地址`
- `祈福语`

### 导入往生数据时显示

- `牌位主体`
- `阳上`
- `年龄`
- `生日`
- `忌日`
- `寄语`
- `地址`
- `亡者信息`

禁止出现“导入延生数据但右侧显示往生字段表”这类状态错位。

## 新建牌位表单字段补全

第一阶段把录入端字段补全纳入正式实施。

### 必补字段

- `属相`
- `年龄`

### 往生表单一并补齐

- `生日`
- `忌日`
- `寄语`

### 默认约定

- 套打字段内部仍沿用现有“祈福语”字段体系做兼容
- 业务录入端如展示“寄语”，映射层需明确对应到套打使用字段

## 数据与打印规则

导入或读取系统数据后，为每条记录补内部字段：

- `__variant`

预览、翻页、打印都按记录自己的 `__variant` 渲染，不允许因翻页或批量数据混合而串版式。

## 实施顺序

1. 落地本文件
2. 整理单张牌位模板配置结构
3. 补新建牌位表单字段
4. 实现导入数据后的自动版式判定
5. 重写模板设计器中的字段切换、画布切换、版式保存
6. 修正右侧数据表字段切换
7. 做回归测试并补验收结论

## Test Plan

- 延生数据：`信人` 有值时自动走版式一；空值时自动走版式二
- 往生数据：`2 亡者` 走版式三；`1 亡者` 且 `年龄/生日/忌日` 全空走版式一；任一有值走版式二
- 切换延生/往生模板时，右侧数据表字段列同步切换
- 切换版式时，字段映射区和画布字段同步切换
- 每个模板每个版式单独保存；关闭后重新打开仍保持正确布局
- 新建牌位表单能录入 `属相 / 年龄 / 生日 / 忌日 / 寄语`，并能进入套打数据源
- 打印预览按每条记录自己的 `__variant` 渲染，不因翻页或批量数据混合而串版式

## 验收口径

第一阶段验收通过的标准：

- 单张牌位主流程只使用 `延生禄位模板` 与 `往生牌位模板`
- 旧的“超度详版 / 简版”不再作为单张主流程模板结构
- 右侧数据表字段不再错位
- 牌位录入字段与套打字段链路一致
- 版式切换、保存、预览、打印四个动作的上下文一致
